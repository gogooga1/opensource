# 1. top - 실시간 프로세스 모니터링

운영 중인 리눅스 시스템의 실시간 프로세스 목록, CPU 사용률, 메모리 사용량 등을 보여주는 모니터링 도구

* **주요 특징**
  * CPU, 메모리 사용량을 실시간 갱신하여 보여줌 (기본 3초)
  * 자원 사용량 높은 순으로 자동 정렬
  * 실행 중 키 입력으로 정렬 방식 변경 등 조작 가능

* **자주 쓰는 단축키** (top 실행 중)
  | 키 | 의미 |
  | :---: | :-------------|
  | q | top 종료 |
  | P | CPU 사용량 정렬 |
  | M | 메모리 사용량 정렬 |
  | k | 프로세스 종료 ***(kill)*** |
  | h | 도움말 표시 |

* **사용 예시**
  
  ```bash
  top
  ```

# 2. ps - 프로세스 상태 출력

현재 실행 중인 프로세스를 스냅샷 형태로 출력하는 명령어

* **주요 특징**
  * 특정 프로세스의 PID 확인 가능
  * 다양한 옵션을 조합해 상세한 프로세스 정보를 출력
  * grep 과 조합해 특정 프로그램 검색 가능

* **자주 쓰는 옵션**
  | 옵션 | 의미 |
  | :------: | :------------------ |
  | ps | 현재 쉘의 프로세스만 표시 |
  | ps -e | 시스템의 모든 프로세스 표시 |
  | ps -f | 풀 포맷 (부모 PID 등 포함) |
  | ps aux | 가장 널리 쓰이는 전체 프로세스 상세 보기 |

* **사용 예시**

  ```bash
  ps aux
  ps aux | grep nginx
  ps -ef | grep java
  ```

# 3. jobs - 백그라운드 작업 목록 확인

현재 터미널에서 실행한 백그라운드 작업(job) 목록을 관리하는 명령어

* **주요 특징**
  * 백그라운드에서 실행한 작업 상태 조회
  * job 번호로 작업 제어 가능
  * 같은 쉘에서 실행한 작업만 관리

* **자주 쓰는 옵션**
  | 옵션 | 의미 |
  | :------: | :------------------ |
  | jobs | 현재 백그라운드 작업 출력 |
  | jobs -l | PID 포함하여 상세하게 표시 |
  | jobs -r | 실행 중인 작업만 표시 |
  | jobs -s | 일시정지(suspended)된 작업만 표시 |

* **사용 예시**

  ```bash
  sleep 100 & # 예시용 sleep 백그라운드 프로그램 100초 실행
  jobs
  jobs -l
  ```

# 4. kill - 프로세스 종료 시그널 보내기

특정 프로세스에 종료 또는 제어 신호(signal)를 보내는 명령어

* **주요 특징**
  * PID 또는 job 번호로 프로세스에 신호 전달
  * 정상 종료 또는 강제 종료 가능
  * 다양한 시그널 사용 가능(SIGTERM, SIGKILL 등)

* **자주 쓰는 옵션/시그널**
  | 옵션/시그널 | 의미 |
  | :------: | :------------------ |
  | kill PID | 기본 종료 요청 (SIGTERM = 15) |
  | kill -9 PID | 강제 종료 (SIGKILL) |
  | kill -STOP PID | 프로세스 일시정지 |
  | kill -CONT PID | 일시정지된 프로세스 재개 |
  | kill %1 | job 번호 1번 kill |

* **사용 예시**

  ```bash
  kill 1234
  kill -9 1234
  kill %1
  ```
